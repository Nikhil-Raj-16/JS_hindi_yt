How JS code runs?
1. Memory creation Phase
2. Execution Phase


1. Global EC -> allocated to this

2. Memory creation Phase -> saare variables ko rkha jata hai, and value is assigned as undefined.
e.g val1 and val2 = undefined, Function like addnum => defination
result1 and result2 = undefined

3.Execution Phase - we take first val1 and put the value in it and same for val2
val1 = 10, val2 = 5

Function ke andar kya hoga that don't come under Execution Phase, so we skip it
now for result1 which calls the Function a new EC is created,i.e (new variable environment + Execution thread)

every time a function is called a new EC is created.    aur use Memory Phase + Execution Phase

and the EC created by function call is deleted after it's Execution.

and now after the above steps we get result1 = 15, which we have in Execution Phase
and same for result2 = 12
